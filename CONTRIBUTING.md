# Contributing to Multimodal RAG Portfolio

Thank you for your interest in contributing!

## Getting Started

1.  Fork the repository.
2.  Clone your fork: `git clone https://github.com/YOUR_USERNAME/multimodal-rag-portfolio.git`
3.  Install dependencies: `make setup`
4.  Create a branch: `git checkout -b feature/my-new-feature`

## Development Standards

- **Code Style**: We use `black` for formatting and `flake8` for linting. Run `make lint` before committing.
- **Tests**: Add unit tests for new features. Run `make test` to verify.
- **Type Hints**: Use Python type hints for all function signatures.

## Pull Request Process

1.  Ensure all tests pass.
2.  Update documentation if necessary.
3.  Open a Pull Request with a clear description of changes.

### Pull Request Template

**Description**
Please include a summary of the change and which issue is fixed.

**Type of change**
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

**Checklist**
- [ ] My code follows the style guidelines of this project
- [ ] I have performed a self-review of my own code
- [ ] I have added tests that prove my fix is effective or that my feature works
- [ ] New and existing unit tests pass locally with my changes
